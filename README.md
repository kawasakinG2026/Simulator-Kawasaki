
# Simulator
## <概要>
　Unityで、「もし人型のメカ兵器があったら」「このようなゲームがやってみたいけど、どこにもないから作ってみよう」という発想をもとに、フライトスティックやペダルなどを用いたシミュレータを制作しています。制作物の提出にあたり、作成したC#のスクリプトを一部抜粋して掲載しています。
　使用したソフトはUnityとBlenderです。Blenderで作成したモデルをUnityへ転送し、アーマチュアの構造を活用して動作の制御を行っています。

# 生成AIツールの使用について
本制作では、ChatGPTを以下の目的で使用しています。

・Unityの基本設定や操作方法の確認
（インプットシステムの設定・背景画像の適用方法など）

・C#の基礎学習、スクリプト作成の補助（ベースコードの生成を含みます）

・挙動改善のための、複雑な計算式作成の補助（[DualStickLook.cs]のvoid FindTarget以降が該当します）

・制作初期におけるエラーやバグの原因特定の補助

#### 一方で、ゲーム制作における設計思想やアイデア、モデル作成、スクリプトの基礎設計や挙動の調整、パラメータの設計などにつきましては自身で行っています。また、READMEと各スクリプトの説明も自身で記述しています。

### <工夫した点・要素>
1.フライトスティック・ペダルによる入力の統合
・本来一つのゲームでは同時に使用しないデバイスを、シミュレータとして流用し、一般的なゲームパッドとは異なる独自の操作方法を実現させました。
・今持っている物・アイデアを最大限に活用しました。

2.移動方向をベクトルの合成によって計算
・二基のフライトスティックによって、それぞれの入力を単純な値ではなくベクトルへ変換し、モデルの各部のブースターによって動いている、という挙動を再現しました。（DualStickLook.cs参照）

3.独自のエイムアシストの設計
・頭部や腕部などの一部のみをターゲット方向へ追従させることにより、人型でありながら機械的な挙動の再現をしました。
・モデルごとのアーマチュア構造に対応するため、ボーン角度の計算式では、座標軸や符号などに汎用性を持たせました。

4.Unityで制作するにあたって必要となるモデルの作成
・ゲーム制作やC#スクリプトの作成のノウハウだけでなく、Blenderによるモデリングやアーマチュア構造への知識も獲得しました。
・リアルな挙動の再現だけでなく、視覚的な楽しさなどの追求なども行っています。


## <スクリプト説明>
　基本的に、各スクリプト内で挙動の説明を行っておりますが、ここでは実際にUnityで使用していくにあたって、各スクリプトの概要と運用方法そのものについての説明を行います。（アルファベット順）

### [AimAssist.cs]
　頭部や腕部を自動的に敵オブジェクトへ追従させることで、プレイヤーは移動に集中しつつも、敵へのエイムを行えるようにするスクリプトです。頭部では、威力が少ないながらもクールタイム無く連射が可能なバルカンを、腕部ではクールタイム有りですが火力の高い主兵装を射撃することで、役割を分けることができます。

　運用方法は、使用したいモデルの親（モデルの中で最も上にある主となるオブジェクト）へアタッチ（インスペクタへ転送）することで使えます。その後、インスペクタ内で適宜設定していくことで動かせるようになります。下の画像は、実際に使用した際の例です。（現在のバージョンとは表記が異なる可能性がありますが、基本的な操作性や手順に変更はありません。）<img width="192" height="337" alt="スクリーンショット (12)" src="https://github.com/user-attachments/assets/d01d7dc6-829b-4f74-84f6-8cd21fd1208f" />

　注意点は、モデルによっては腕が意図しない方向へ向く場合があることです。これは、アーマチュアの構造の問題で、モデルの中心となるボーンと、腕のボーンの向きが違うことが原因であると考えられます。例えば、腹部からアーマチュアを作成した場合、中心と腕のボーンの向きは恐らくですが真逆の向きになると考えられます。

対処法は、スクリプト内の115,116行での角度の計算で、

 float pitch = Mathf.Clamp【(Mathf.Asin(-localDir.x)】 * Mathf.Rad2Deg, minPitch, maxPitch);
 
 float yaw   = Mathf.Atan2【(-localDir.y,localDir.z)】* Mathf.Rad2Deg;

の中の【】内の符号やx,y,zをそれぞれ置き換えてみることです。この問題が解決できれば、その後のオフセットや回転制限なども設定しやすいです。なお、UIの設定は任意で行えるようにしており、私の場合は空欄のままにしてあります。

### [Bullet.cs]
　弾丸のダメージと処理を行っています。ワールド内に弾が残らないように、生存時間を設定することができます。また、スクリプト内23行の<>の中にあるスクリプトと連動してダメージ計算を行っています。そのため、<>内のスクリプトは、名前を一致させる必要があります（今回の場合は[EnemyHP]）。また、コリジョンによる当たり判定を用いて、ダメージ計算を行っています。

　運用方法は、弾丸オブジェクトそのものにアタッチすることで効果が付与されます。注意点として、コリジョンとリジットボディも設定しなければ挙動が不安定になります。リジットボディでは、[Is Kinematic]のチェックを外し、[Use Gravity]のチェックを適宜変更することを推奨します。私のプロジェクトでは、ワールドが宇宙空間であるのを想定して制作しているため、Use Gravityのチェックは外してあります。

### [DualStickLook.cs]
　二基のフライトスティックを、操縦桿として流用することで視点操作を行うスクリプトです。左スティックを↓、右スティックを↑に倒すと左（←）に回転し、両方前（↑）に倒すと操作しているモデルが前へ倒れるような挙動をとります。スティックを横に倒すと、その場合のみ横へ移動します。感覚として、モデルの各部位にブースターが付いてあるのを想定し、スティックを倒した方向にブースターが噴射される、というのを想像すると考えやすいです。そのため、計算式が単なる移動の量ではなく、ベクトルの合計で求められているため複雑となっています。
 
　運用方法は[AimAssist.cs]と同じく親モデルへアタッチすることで使用可能です。インプットアクションはインスペクタ内で設定できるようになっています。なお、私のプロジェクトとスクリプトでは、全てそれぞれのインスペクタでインプットアクションを適宜設定していく方式をとっています。

　例として、使用したコントローラーは、左スティックが[HID::Thrustmaster T.Flight Stick X]、右スティックが[HID::Thrustmaster T.Flight Stick Hotas X]です。このスクリプトの操作であれば、ゲームパッドのスティックでも同じ挙動を取ることが可能です。

### [EnemyHP.cs]
　自分が発射した弾丸と、敵オブジェクトへのダメージ計算を行うスクリプトです。拡張機能として、敵オブジェクトのHPが0になった場合に、撃破エフェクトが発生する機能を設定できるようになっています。運用方法は、敵オブジェクトの親にアタッチし、同時にコリジョンも設定しておくことで使えます。インスペクタでの設定は比較的シンプルで、HPの設定と撃破エフェクトのみです。エフェクトは設定しなくてもゲームの進行に影響はありません。

### [FireAndFlash.cs]
　射撃スクリプトです。インスペクタで弾丸のオブジェクト、発射口、射速、マズルフラッシュのエフェクトなどを設定することができます。
　運用方法は、インプットアクションは[DualStickLook.cs]と同様に、適宜インスペクタで設定することでボタンを選択できます。銃の向きと弾丸が発射される方向が異なる場合は、[Fire Point]にアタッチした空オブジェクトの角度を変えることで改善できます。

### [PedalMovement.cs]
　ドライブシミュレータのペダルを流用して、前後への移動を行えるようにしたスクリプトです。[DualStickLook.cs]と併用することで、三次元的な移動を可能としています。
　運用方法は、親モデルへアタッチしてインプットアクションなどを設定するため、[DualStickLook.cs]と使い方は同じです。使用したペダルは、[HID::Logitec G923 Racing Wheel for PlayStation 4 and PC]です。なお、試験的にゲームパッドでの操作を行いましたが、スクリプトの構造がペダルを前提としたものであるため、[DualStickLook.cs]のようにパッドでも問題なく動くわけではないため、あまり推奨しません。（インプットアクションのコントロールタイプが、ペダルとコントローラーでは異なるのが原因の一つであると考えられます。）

### [Vulcan.cs]
　[FireAndFlash.cs]と同様に射撃スクリプトですが、こちらはフルオートでの操作が可能となっています。運用方法も[FireAndFlash.cs]と同じく、インスペクタでの設定をすることになりますが、こちらでは移動方向補正という機能が追加されています。これは、[AimAssist.cs]での説明で述べたように、腕部の主兵装との差別化を図っている要素の一つです。具体的には、今後追加していく「敵モデルの部位破壊」や「追尾ミサイルの撃墜補助」などを想定して、副兵装であるバルカンの有用性を向上させるためにあります。
